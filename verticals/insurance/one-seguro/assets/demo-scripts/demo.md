## Demo Flow

### Opening (2 minutes)

**Script:**
"Today I want to show you how One Seguros is using Cognigy to modernize their claims process. Instead of customers calling and waiting on hold, they can now report a claim in two ways:

1. **Voice** - Speak naturally with our AI agent
2. **Web** - Fill a smart form on their phone or computer

Both options work 24/7 and reduce processing time from hours to minutes."

---

### Demo 1: Voice Claims Report (7 minutes)

#### Setup
"Let me start with the voice scenario. Imagine a customer had a car accident and wants to report it immediately."

#### Interaction
1. **Show opening screen**
   - Agent greeting appears
   - Natural, friendly tone

2. **User selects voice option**
   - "I want to report a claim"

3. **Agent asks questions naturally**
   - "What type of claim?"
   - "When did it happen?"
   - "Any injuries?"
   - "Location?"
   - "Other people involved?"

4. **Highlight key features as they appear**
   - "Notice how the agent asks follow-up questions contextually"
   - "It's understanding natural language, not keywords"
   - "It's confirming information to avoid errors"

5. **Confirmation**
   - Show claim number generated
   - Email confirmation sent
   - Claim appears in system

**Script highlight:**
"What's powerful here is the AI understands context. It knows a car accident might mean injuries, so it asks. It doesn't ask the same question twice. And crucially, it confirms everything before submitting."

---

### Demo 2: Web Claims Report - XApp (6 minutes)

#### Setup
"Now let's look at the web experience. Maybe a customer prefers a form, or needs to upload photos."

#### Interaction
1. **Show landing page**
   - Professional interface
   - Clear steps (Step 1 of 5)

2. **Step 1: Claim Type**
   - "They select the type of claim"
   - Show dropdown/buttons
   - Show you can ask questions about the page

3. **Step 2: Incident Details**
   - "The form adapts to the claim type"
   - Shows relevant fields only
   - Date picker, text inputs

4. **Step 3: Third Party Info**
   - "If it's an accident, it asks for third party details"
   - Conditional logic in action

5. **Step 4: Document Upload**
   - "The game changer: immediate photo upload"
   - Drag and drop demo
   - Show file validation

6. **Step 5: Review**
   - "Final review before submission"
   - All info summarized
   - Option to edit

7. **Confirmation**
   - "Instant claim number"
   - "Real-time status tracking"
   - "Email + SMS notifications"

**Script highlight:**
"The XApp is mobile-responsive, so customers can report claims from anywhere. And because it's built in Cognigy, we get real-time integrations with the backend systems."

#### Mention English
"We also support English for international customers."

**Script highlight:**
"Supporting multiple languages without building multiple systems - that's the power of Cognigy."

## Q&A Scenarios

### Q: "Can it handle complex claims?"
**A:** "Great question. For simple claims like 'my phone screen broke', it handles end-to-end. For complex ones - like multiple injuries, large damage - it intelligently escalates to a human agent who already has all the context."

### Q: "What about security?"
**A:** "All data is encrypted in transit and at rest. The system complies with local data protection regulations."

### Q: "What's the cost?"
**A:** "Cognigy operates on a per-conversation basis."

### Q: "Can we customize this?"
**A:** "Absolutely. This is a template we've refined, but every client has unique requirements. We build on top of this foundation."

## Demo Tips & Tricks

### If Voice Demo Fails
- Have a pre-recorded video of the voice interaction
- Show the conversation logs instead
- Explain: "Same technology, we just showed the logs"

### If XApp Won't Load
- Use the screenshots folder
- Narrate through the flow
- Still impactful

### For Remote Demos
- Share screen using Zoom/Teams
- Slow down your typing (audiences can't follow fast typing)
- Pause after each action (let it sink in)

### Engagement Tips
- Ask questions: "What would you do if...?"
- Use pauses for effect
- Show the human connection (escalation to real agent)
- Talk benefits, not features

### Time Management
- 15 min version: Skip admin features
- 20 min version: Include analytics
- 30 min version: Live Q&A + deep dive on one feature
